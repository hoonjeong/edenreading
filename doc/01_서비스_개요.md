# 이든독서 교육원 학생 관리 시스템 - 서비스 개요

## 1. 서비스 정의

| 항목 | 내용 |
|------|------|
| **서비스명** | 이든독서 교육원 학생 관리 시스템 (가칭: 이든톡 / EdenTalk) |
| **목적** | 재원 학생의 학습·독서 활동을 체계적으로 기록하고, 학부모와 안전하게 공유하는 교육원 전용 관리 플랫폼 |
| **타겟 사용자** | (1차) 교육원 원장·선생님, (2차) 재원 학생의 학부모 |
| **핵심 가치** | 학생의 성장 과정을 시각적으로 기록 → 학부모의 신뢰 향상 → 재원 유지율 상승 |

## 2. 사용자 그룹 정의

### 가. 관리자 (원장 / 선생님)
- **역할**: 학생 활동 기록, 시험 출제·채점, 학부모 소통, 상담·출석 관리
- **사용 환경**: 주로 PC (수업 후 기록), 일부 모바일 (현장 사진 업로드)
- **핵심 니즈**: 빠르고 간편한 기록, 학부모와의 효율적 소통, 학생별 성장 추이 파악

### 나. 학부모
- **역할**: 자녀의 활동 열람, 시험 리포트 확인, 메시지 수신, 상담 신청
- **사용 환경**: 주로 모바일 (수시 확인)
- **핵심 니즈**: 자녀가 무엇을 어떻게 배우는지 시각적으로 확인, 성장 과정 추적

### 다. 학생 (간접 사용자)
- 직접 로그인하지 않으나, 모든 기록의 주체
- 학부모 계정과 1:N 또는 N:1 연동 (한 학부모가 여러 자녀, 한 자녀에 여러 보호자 가능)

## 4. 핵심 차별 포인트

1. **학생-학부모 데이터 격리**: 한 학부모는 본인 자녀의 정보만 볼 수 있도록 엄격한 권한 분리
2. **자동 채점 + 리포트**: 정답을 미리 입력해두면 응시 결과 입력 시 자동 채점 → 회차별·영역별 분석 리포트 자동 생성
3. **멀티미디어 활동 기록**: 사진·영상·글을 자유롭게 조합하여 학생 활동을 풍부하게 기록
4. **인증코드 가입**: 무분별한 학부모 가입을 방지하고 교육원이 발급한 코드로만 가입 가능

## 5. 기술적 고려사항 (기획 관점)

| 영역 | 고려사항 |
|------|---------|
| **반응형** | 학부모는 모바일 우선, 관리자는 PC 우선 — 두 환경 모두 최적화 필요 |
| **알림** | 카톡 알림톡 또는 SMS 연동 (활동/시험 업로드 시 학부모 알림) |
| **저장소** | 사진·영상 업로드를 고려한 충분한 미디어 스토리지 |
| **보안** | 학생 개인정보 보호, 학부모 간 정보 격리 (Row Level Security) |
| **확장성** | 향후 다른 교육원으로의 확장 가능성 고려한 멀티테넌시 구조 검토 |

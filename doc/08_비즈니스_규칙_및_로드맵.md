## 2. 단계별 구현 로드맵

서비스의 안정적 출시를 위한 4단계 구현 계획입니다.

### Phase 1: MVP - 핵심 기능 (1~2개월)

**목표**: 가장 핵심적인 활동 기록·공유 기능으로 빠르게 베타 출시

| 영역 | 포함 기능 |
|------|----------|
| 회원 관리 | 관리자 가입·로그인, 학부모 등록·인증코드 가입·로그인 |
| 학생 관리 | 학생 등록, 학부모 연동 |
| 활동 관리 | 주간 활동 기록(사진+글), 학부모 자동 공유, 알림(SMS) |
| 학부모 페이지 | 자녀 활동 열람 |

**검증 포인트**: 학부모가 카톡/SMS로 알림을 받고 자녀 활동을 모바일에서 부드럽게 볼 수 있는가

---

### Phase 2: 핵심 기능 완성 (1~2개월)

**목표**: 출석·독서·메시지를 추가하여 일상 운영이 가능한 수준으로 완성

| 영역 | 포함 기능 |
|------|----------|
| 활동 관리 | 영상 업로드 추가 |
| 출석 관리 | 일별 출결 체크, 학부모 알림 |
| 독서 기록 | 책 등록, 활동 사진/소감 공유 |
| 메시지 | 학부모 단체/개별 메시지 발송 (카톡 알림톡 + SMS) |
| 학부모 페이지 | 출석 캘린더, 독서 목록, 알림/메시지 센터 |

**검증 포인트**: 교육원 운영자가 하루 종일의 모든 학생 운영 정보를 시스템 안에서 처리할 수 있는가

---

### Phase 3: 시험·리포트 (2~3개월)

**목표**: 차별화 기능인 자동 채점 및 리포트로 서비스 고도화

| 영역 | 포함 기능 |
|------|----------|
| 시험 관리 | 회차 생성, 문항/정답 입력, 응시자 배정 |
| 자동 채점 | 답안 입력 시 즉시 채점 |
| 리포트 | 학생별 종합/영역별/추이 그래프 |
| 학부모 페이지 | 시험 리포트 열람, 회차별 비교 |
| 분석 | 관리자용 종합 리포트(반 평균, 문항별 정답률) |

**검증 포인트**: 30명 학생 답안 입력에서 학부모 리포트 공유까지 30분 내 완료 가능한가

---

### Phase 4: 운영 고도화 (2개월~)

**목표**: 상담·확장성·운영 편의성 강화

| 영역 | 포함 기능 |
|------|----------|
| 상담 관리 | 학부모 상담 신청, 일정 관리, 상담 기록 누적 |
| 통계 | 학생별 종합 성장 리포트, 출결·시험 통합 분석 |
| 권한 관리 | 선생님별 담당 학생 분리, 세분화된 권한 |
| 멀티 교육원 | 향후 다른 교육원으로 확장 위한 멀티테넌시 구조 |
| 모바일 앱 | 학부모용 네이티브 앱 (Push 알림 강화) |

**검증 포인트**: 신규 교육원이 1주일 내에 자체적으로 시스템 셋업·운영 가능한가

## 3. 기획 단계 우선순위 결정 매트릭스

기능별 우선순위를 사용자 가치 vs 구현 난이도로 정리하여 의사결정에 활용합니다.

| 기능 | 사용자 가치 | 구현 난이도 | Phase |
|------|------------|------------|-------|
| 학부모 인증코드 가입 | 높음 | 낮음 | 1 |
| 주간 활동 기록·공유 | 높음 | 낮음 | 1 |
| 출석 관리 | 높음 | 낮음 | 2 |
| 독서 기록 | 중간 | 낮음 | 2 |
| 카톡 알림톡 발송 | 높음 | 중간 | 2 |
| **시험 자동 채점·리포트** | **매우 높음** | **높음** | **3** |
| 상담 관리 | 중간 | 중간 | 4 |
| 멀티테넌시 | 낮음(현재) | 매우 높음 | 4+ |

## 4. 다음 단계 제안

기획 문서가 완성된 이후 진행할 작업 순서:

1. **데이터베이스 설계 (ERD)**: 본 기획서를 기반으로 학생/학부모/활동/시험/리포트 등 엔티티 관계도 작성
2. **API 명세**: 화면별로 필요한 API 엔드포인트 정의
3. **와이어프레임 → 디자인 시안**: 화면정의서 기반으로 UI 디자인
4. **MVP 개발 착수**: Phase 1 범위 확정 후 4~6주 스프린트
5. **베타 테스트**: 이든독서 교육원 내부 1~2주 테스트, 피드백 반영

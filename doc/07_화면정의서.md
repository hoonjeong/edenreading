# 화면 정의서

각 화면(페이지)의 목적, 사용자 행동, 주요 구성 요소, 다음 단계, 예외 처리를 정리합니다.
실제 디자인·개발 단계에서 와이어프레임의 기반이 됩니다.

---

## A. 공통 화면

### A-1. 서비스 진입 / 로그인 선택 화면
| 항목 | 내용 |
|------|------|
| **목적** | 사용자가 자신이 누구인지(관리자/학부모) 선택하고 로그인 흐름으로 진입 |
| **사용자 행동** | 두 가지 진입 버튼 중 선택 |
| **주요 구성** | 서비스 로고, "관리자 로그인" 버튼, "학부모 로그인" 버튼, 가입 안내 문구 |
| **다음 단계** | 선택한 사용자 유형의 로그인 화면 |
| **예외 처리** | 학부모가 관리자 화면으로 잘못 진입 시, 로그인 실패 후 안내 문구 노출 |

---

## B. 관리자 영역 화면

### B-1. 관리자 회원가입
| 항목 | 내용 |
|------|------|
| **목적** | 신규 관리자(원장 또는 선생님)가 시스템에 가입 |
| **사용자 행동** | 본인 정보 입력 후 가입 신청 |
| **주요 구성** | 이름, 이메일, 휴대폰, 비밀번호, 권한 유형(원장/선생님), 약관 동의 |
| **다음 단계** | 가입 완료 후 자동 로그인 → 대시보드 |
| **예외 처리** | 이메일 중복 시 안내, 권한 유형이 "원장"인 경우 첫 등록만 자동 승인, 추가 원장은 기존 원장 승인 필요 |

### B-2. 관리자 로그인
| 항목 | 내용 |
|------|------|
| **목적** | 등록된 관리자의 시스템 진입 |
| **사용자 행동** | 이메일·비밀번호 입력 |
| **주요 구성** | 이메일, 비밀번호, "로그인 유지" 체크박스, "비밀번호 찾기" 링크 |
| **다음 단계** | 대시보드 |
| **예외 처리** | 5회 연속 실패 시 30분 잠금 + 본인 인증 안내 |

### B-3. 관리자 대시보드
| 항목 | 내용 |
|------|------|
| **목적** | 오늘의 핵심 정보를 한눈에 파악하고 빠른 작업 진입 |
| **사용자 행동** | 알림 확인, 핵심 메뉴로 이동 |
| **주요 구성** | (1) 오늘의 알림(상담 요청/메시지 수신 등), (2) 출석 미입력 알림, (3) 미공유 활동 카드, (4) 빠른 메뉴 카드 (활동 기록/시험 관리/메시지 등), (5) 학생 검색 바 |
| **다음 단계** | 클릭한 메뉴/알림에 따른 상세 화면 |
| **예외 처리** | 신규 가입 직후 데이터 없음 → "학생을 먼저 등록하세요" 온보딩 안내 |

### B-4. 관리자 회원 관리
| 항목 | 내용 |
|------|------|
| **목적** | 시스템을 사용하는 관리자(선생님 등) 추가/수정/삭제 |
| **사용자 행동** | 관리자 목록 조회, 추가, 권한 수정, 삭제 |
| **주요 구성** | 관리자 목록 테이블(이름/권한/가입일/최근 접속), "관리자 추가" 버튼, 행별 수정·삭제 버튼 |
| **다음 단계** | 추가/수정 폼 모달 |
| **예외 처리** | 자기 자신 삭제 불가, 마지막 원장 권한 삭제 불가 |

### B-5. 학부모 등록
| 항목 | 내용 |
|------|------|
| **목적** | 신규 학부모를 시스템에 등록하고 가입 인증코드 발급 |
| **사용자 행동** | 학부모 정보 입력 → 인증코드 자동 생성 → 발송 방식 선택 |
| **주요 구성** | 이름, 휴대폰, 이메일(선택), 메모 / 생성된 6자리 인증코드 / 발송 버튼(카톡, SMS, 코드 복사) |
| **다음 단계** | 학부모 목록 화면 또는 "학생 연동" 화면 |
| **예외 처리** | 동일 휴대폰 번호 중복 시 "기존 학부모 확인" 안내, 알림톡 발송 실패 시 SMS 자동 전환 |

### B-6. 학부모 목록
| 항목 | 내용 |
|------|------|
| **목적** | 등록된 학부모 조회 및 관리 |
| **사용자 행동** | 검색, 필터(가입 완료/미완료), 상세 조회, 수정·삭제 |
| **주요 구성** | 학부모 목록 테이블(이름/휴대폰/연동 자녀/가입 상태), 필터, 검색 |
| **다음 단계** | 학부모 상세 화면 |
| **예외 처리** | 가입 미완료 학부모는 "코드 재발급" 버튼 노출 |

### B-7. 학생 등록
| 항목 | 내용 |
|------|------|
| **목적** | 신규 학생 등록 및 학부모 연동 |
| **사용자 행동** | 학생 정보 입력 → 학부모 검색 → 연동 |
| **주요 구성** | 이름, 학년, 생년월일, 특이사항, 사진 / 학부모 검색·다중 연동(아빠+엄마 등 N:1) |
| **다음 단계** | 학생 목록 또는 학생 상세 화면 |
| **예외 처리** | 학부모 미선택 시 경고, 동명이인 학부모 선택 실수 방지 위해 휴대폰 뒷자리 표시 |

### B-8. 학생 상세 카드
| 항목 | 내용 |
|------|------|
| **목적** | 한 학생에 대한 모든 정보를 통합 조회 |
| **사용자 행동** | 활동·시험·독서·출석·상담 이력 확인, 빠른 기록 추가 |
| **주요 구성** | 학생 기본 정보, 연동 학부모, 탭(활동/시험/독서/출석/상담), 이력별 카드 목록 |
| **다음 단계** | 각 이력의 상세 또는 신규 기록 추가 화면 |
| **예외 처리** | 데이터 없음 → "첫 활동을 기록해보세요" 안내 |

### B-9. 주간 활동 기록
| 항목 | 내용 |
|------|------|
| **목적** | 학생의 주간 활동을 사진·영상·글로 기록하고 학부모에게 공유 |
| **사용자 행동** | 학생 선택 → 콘텐츠 업로드 → 공유 여부 선택 |
| **주요 구성** | 학생 선택 드롭다운, 일자, 제목, 본문, 사진·영상 업로더(드래그앤드롭), "임시 저장"/"공유" 버튼 |
| **다음 단계** | 활동 카드 등록 완료 → 학부모 자동 알림 |
| **예외 처리** | 영상 용량 제한 초과 시 안내, 업로드 중 이탈 시 임시 저장, 학부모가 미연동 학생일 경우 공유 비활성화 |

### B-10. 시험 회차 생성·관리
| 항목 | 내용 |
|------|------|
| **목적** | 신규 시험 회차를 생성하고 문항/정답을 입력 |
| **사용자 행동** | 시험 정보 → 문항 정보 → 응시자 배정 |
| **주요 구성** | 시험명, 일자, 문항 수, 문항별 (번호/유형/배점/정답) 입력 테이블, 응시 학생 다중 선택 |
| **다음 단계** | 답안 입력 화면 |
| **예외 처리** | 문항 수와 입력 행 수 불일치 시 경고, 정답 미입력 문항 표시 |

### B-11. 답안 입력 및 자동 채점
| 항목 | 내용 |
|------|------|
| **목적** | 학생별 답안을 빠르게 입력하고 즉시 채점 결과 확인 |
| **사용자 행동** | 학생 선택 → 답안 1번부터 입력 → 자동 채점 결과 확인 |
| **주요 구성** | 학생 선택 드롭다운, 1~N번 답안 입력 셀, "채점" 버튼, 즉시 결과 표시(점수/영역별 정답률) |
| **다음 단계** | 다음 학생 또는 리포트 화면 |
| **예외 처리** | 미입력 문항은 오답 처리 + 경고 표시, 입력 중 화면 이동 시 자동 임시 저장 |

### B-12. 시험 리포트 (관리자용)
| 항목 | 내용 |
|------|------|
| **목적** | 시험 결과를 분석·검토하고 학부모에게 공유 |
| **사용자 행동** | 학생별 또는 전체 리포트 조회, 코멘트 작성, 공유 |
| **주요 구성** | 학생별 점수 그래프, 영역별 정답률, 회차별 추이, 문항별 정답률(전체), 코멘트 입력란, "학부모 공유" 버튼 |
| **다음 단계** | 학부모 알림 발송 |
| **예외 처리** | 미채점 응시자 존재 시 공유 차단 + 안내 |

### B-13. 독서 기록 등록
| 항목 | 내용 |
|------|------|
| **목적** | 학생이 읽은 책과 관련 활동을 기록 |
| **사용자 행동** | 학생 선택 → 책 정보 입력 → 활동 사진/소감 추가 |
| **주요 구성** | 학생 선택, 책 제목·저자·표지 사진, 읽은 날짜, 활동 사진, 선생님 소감 |
| **다음 단계** | 학부모 자동 공유 |
| **예외 처리** | 표지 사진 미첨부 시 기본 이미지 자동 사용 |

### B-14. 메시지 발송
| 항목 | 내용 |
|------|------|
| **목적** | 학부모에게 단체 또는 개별 메시지 발송 |
| **사용자 행동** | 수신 대상 선택 → 채널 선택 → 메시지 작성 → 발송 |
| **주요 구성** | 수신 대상 검색·선택(개별/반/전체), 채널 선택(카톡/SMS/앱푸시), 메시지 입력란, 발송 예약, 미리보기 |
| **다음 단계** | 발송 결과 화면 |
| **예외 처리** | 카톡 알림톡 템플릿 미일치 시 SMS로 자동 전환 |

### B-15. 상담 관리
| 항목 | 내용 |
|------|------|
| **목적** | 상담 요청 접수, 일정 확정, 내용 기록 |
| **사용자 행동** | 신규 요청 확인 → 일정 확정 → 상담 후 기록 저장 |
| **주요 구성** | 신규/예정/완료 탭, 상담 카드(학부모/학생/희망일/상태), 상담 기록 작성 폼 |
| **다음 단계** | 학생 카드에 누적 표시 |
| **예외 처리** | 일정 중복 시 경고 |

### B-16. 출석 관리
| 항목 | 내용 |
|------|------|
| **목적** | 일별 출결 체크 및 통계 조회 |
| **사용자 행동** | 일자 선택 → 학생별 체크 → 저장 |
| **주요 구성** | 일자 선택, 학생 목록 + 출결 토글(출석/지각/결석/사유), "전체 출석으로 일괄 체크" 버튼, 월별 캘린더 통계 |
| **다음 단계** | 저장 시 학부모 자동 알림 |
| **예외 처리** | 미체크 학생 존재 시 저장 시 안내 |

## C. 학부모 영역 화면

### C-1. 학부모 가입 (인증코드 입력)
| 항목 | 내용 |
|------|------|
| **목적** | 교육원이 발급한 인증코드로만 가입 가능하도록 제한 |
| **사용자 행동** | 6자리 인증코드 입력 → 정보 확인 → 비밀번호 설정 |
| **주요 구성** | 인증코드 입력 필드, 등록된 본인 정보(이름/휴대폰 일부) 확인, 비밀번호 설정, 약관 동의 |
| **다음 단계** | 가입 완료 후 자동 로그인 → 학부모 홈 |
| **예외 처리** | 코드 무효/만료/이미 사용됨 → 각각 다른 안내 + "교육원 문의" 버튼 |

### C-2. 학부모 로그인
| 항목 | 내용 |
|------|------|
| **목적** | 등록된 학부모의 시스템 진입 |
| **사용자 행동** | 휴대폰 번호 + 비밀번호 입력 |
| **주요 구성** | 휴대폰, 비밀번호, "로그인 유지", "비밀번호 찾기" |
| **다음 단계** | 학부모 홈 |
| **예외 처리** | 5회 연속 실패 시 잠금, SMS 본인 인증 후 재설정 |

### C-3. 학부모 홈
| 항목 | 내용 |
|------|------|
| **목적** | 자녀의 최신 활동·알림을 빠르게 확인 |
| **사용자 행동** | 자녀 카드 확인, 최신 알림 탭 |
| **주요 구성** | 자녀 카드(여러 명일 시 좌우 스와이프), 최신 활동 미리보기 3개, 새 알림 배지, 빠른 메뉴(활동/시험/독서/출석/상담) |
| **다음 단계** | 선택한 카드/메뉴의 상세 화면 |
| **예외 처리** | 첫 가입 시 활동 데이터 없음 → "선생님이 곧 첫 활동을 공유할 예정입니다" 안내 |

### C-4. 자녀 활동 보기
| 항목 | 내용 |
|------|------|
| **목적** | 자녀의 주간 활동을 시간 순으로 열람 |
| **사용자 행동** | 활동 카드 탭 → 사진/영상/글 확인 |
| **주요 구성** | 활동 카드 목록(최신순), 카드별 썸네일, 상세 화면(슬라이드 사진, 영상 플레이어, 본문 글) |
| **다음 단계** | 상세 화면에서 좋아요/댓글(선택 기능) |
| **예외 처리** | 영상 로딩 실패 시 다운로드 옵션 안내 |

### C-5. 시험 리포트 보기
| 항목 | 내용 |
|------|------|
| **목적** | 자녀의 시험 결과와 성장 추이를 시각적으로 확인 |
